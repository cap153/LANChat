# Web 端使用指南

## Web 端的设计目的

Web 端主要用于以下场景：
1. **无图形界面的服务器**（如你的 Debian 服务器）
2. **临时访问**（不想安装桌面端）
3. **跨平台兼容**（任何有浏览器的设备）

## 文件传输说明

### 工作原理

Web 端的文件传输需要经过两次网络传输：

```
浏览器（你的电脑）
    ↓ 上传文件到 Web 端
Web 端服务器（Debian 服务器）
    ↓ 转发文件到接收方
接收方（局域网其他设备）
```

### 性能对比

| 访问方式 | 文件来源 | 传输次数 | 性能 | 推荐场景 |
|---------|---------|---------|------|---------|
| 本地访问 | 服务器本地 | 1次 | ⭐⭐⭐⭐⭐ | 服务器上直接使用 |
| 局域网访问 | 用户电脑 | 2次 | ⭐⭐⭐ | 临时使用、小文件 |
| 远程访问 | 外网电脑 | 2次 | ⭐ | 仅文本消息 |

### 使用建议

**✅ 推荐使用 Web 端的场景：**
- 发送文本消息
- 发送小文件（< 100MB）
- 临时访问，不想安装桌面端
- 在服务器本地使用浏览器

**❌ 不推荐使用 Web 端的场景：**
- 发送大文件（> 1GB）
- 频繁发送文件
- 需要零拷贝、低内存占用

**💡 大文件传输建议：**
- 使用桌面端（支持零拷贝，直接从硬盘读取）
- 或者在服务器上使用命令行工具直接操作

## 桌面端 vs Web 端

### 桌面端（推荐用于文件传输）

**优点：**
- ✅ 零拷贝，低内存占用
- ✅ 支持任意大小文件
- ✅ 拖拽/粘贴直接获取文件路径
- ✅ 只需一次网络传输

**缺点：**
- ❌ 需要安装
- ❌ 仅支持 Windows、macOS、Linux 桌面

### Web 端（推荐用于消息和小文件）

**优点：**
- ✅ 无需安装
- ✅ 跨平台（任何浏览器）
- ✅ 适合服务器环境

**缺点：**
- ❌ 文件需要上传到服务器
- ❌ 需要两次网络传输
- ❌ 大文件可能导致浏览器卡顿
- ❌ 占用服务器带宽和内存

## 实际使用示例

### 示例 1：在 Debian 服务器上使用

```bash
# 1. 启动 Web 端
./lanchat-web --port 8888

# 2. 在服务器上打开浏览器
firefox http://localhost:8888

# 3. 发送文件
# 文件路径：服务器本地 → 接收方
# 性能：✅ 好（只需一次传输）
```

### 示例 2：从笔记本访问服务器

```bash
# 1. 服务器运行 Web 端
./lanchat-web --port 8888

# 2. 笔记本打开浏览器
http://192.168.1.x:8888

# 3. 发送文件
# 文件路径：笔记本 → 服务器 → 接收方
# 性能：⚠️ 中等（需要两次传输）
# 建议：仅发送小文件
```

### 示例 3：最佳实践

**发送大文件：**
- 使用桌面端（零拷贝）

**发送小文件/消息：**
- 可以使用 Web 端

**服务器环境：**
- 在服务器本地使用 Web 端（性能好）
- 或者使用桌面端（如果有图形界面）

## 技术细节

### Web 端文件上传流程

1. **浏览器端**（`src/js/api.js`）：
   ```javascript
   // 读取文件到内存
   const file = e.target.files[0];
   
   // 分块上传到 Web 端
   const formData = new FormData();
   formData.append('chunk', chunk);
   await fetch(`http://${peerAddr}/api/upload`, {
       method: 'POST',
       body: formData
   });
   ```

2. **Web 端服务器**（`src-tauri/src/web_server.rs`）：
   ```rust
   // 接收分块并保存到临时文件
   async fn upload_file_http() {
       // 保存到 download_dir
       let path = download_dir.join(&file_name);
       file.write_all(&chunk_data).await?;
   }
   ```

3. **转发给接收方**：
   - Web 端再次读取临时文件
   - 通过 HTTP 发送给接收方

### 为什么不能零拷贝？

**浏览器限制：**
- 浏览器无法访问服务器的文件系统
- 必须通过 HTTP 上传文件内容
- 这是 Web 安全模型的基本要求

**解决方案：**
- 使用桌面端（可以零拷贝）
- 或者接受 Web 端的性能限制

## 总结

- Web 端适合**消息和小文件**
- 桌面端适合**大文件传输**
- 在服务器本地使用 Web 端性能最好
- 远程访问 Web 端仅推荐发送消息
